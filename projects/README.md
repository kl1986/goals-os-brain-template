# projects

Active projects: specific outcomes with a goal and, ideally, a deadline — as distinct from an Area's ongoing, no-end-date standard.

**What the Engine/Adapters expect to find here:**

- One folder per project, with a main note carrying its goal, status, next actions, and a dated log of what advanced it.
- Area agents surface next actions from here (and from the Dashboard) when planning with the user.
- Completed or abandoned projects move to `../archive/`.

Empty at template-clone time.
